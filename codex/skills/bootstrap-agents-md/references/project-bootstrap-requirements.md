## 初始化要求：实施“自分形 & 自描述”方法论

在新项目初始化阶段，必须把以下规范写入仓库 `AGENTS.md`，并作为后续开发默认约束。

### 1. 分级加载（自分形）
- 每个业务模块目录都必须提供 `Claude.md`。
- `Claude.md` 至少包含四部分：
  - 地位：该模块在系统中的职责与边界。
  - 逻辑：该模块核心处理流程与关键决策。
  - 约束：调用、依赖、输入输出、性能与安全限制。
  - 业务域清单：该目录下子模块与功能列表。
- 模块越深，说明越具体；上层目录关注边界与编排，下层目录关注实现细节。

### 2. 源码文件三行头部注释（自描述）
每个源码文件头部必须固定包含三行注释（紧贴文件开头）：
- `INPUT:` 依赖什么（外部输入、上游模块、关键配置）
- `OUTPUT:` 提供什么（输出、可复用能力、对外接口）
- `POS:` 在系统中的地位（职责、上下游关系、不可越界范围）

示例：
```text
# INPUT: user profile + billing config
# OUTPUT: normalized subscription snapshot
# POS: billing domain adapter between API and ledger
```

### 3. 自动逐级更新机制
当任一源码发生变更时，必须同步更新三层描述信息：
1. 当前源码文件的三行头部注释。
2. 当前文件所在目录的 `Claude.md`。
3. 上一级目录的 `Claude.md`（必要时继续向上更新，直到描述一致）。

### 4. 执行目标
- 让 AI 在检索时更快命中目标文件与模块。
- 让 AI 在阅读任意命中文件时都能先获得清晰的业务上下文。
- 降低跨层级理解偏差，确保实现与系统边界一致。

### 5. 落地要求
- 新建模块时必须同时创建或补齐对应 `Claude.md`。
- 新增源码文件时必须写入三行头部注释。
- Code Review 需将“描述信息是否同步更新”作为必检项。
